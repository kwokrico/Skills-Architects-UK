# Operational SOPs — UK Architect Master Suite

## Intake checklist (Phase 1)

Before specialist analysis, confirm or request:

- [ ] Project name, site address, and jurisdiction (England/Wales/Scotland/NI)
- [ ] HRB vs non-HRB classification (or flag as unknown)
- [ ] Current RIBA stage and target decision gate
- [ ] Planning status (pre-app, submitted, granted, conditions outstanding)
- [ ] Building Control route (LABC, approved inspector, registered building control approver)
- [ ] Appointment scope (lead designer, PD, contract administrator, site role)
- [ ] Procurement route and form of contract if commercial question

## Escalation paths

| Situation | Escalate to |
|-----------|-------------|
| Statutory interpretation dispute | Client + legal counsel; record in decision log |
| Fire strategy conflict | Fire engineer + Building Control / BSR as applicable |
| Planning condition ambiguity | Planning consultant or LPA clarification |
| Contract variation / claim | Contract administrator + QS; `uk-tender-contract-administration` |
| PI or liability exposure | Practice principal + insurer; `uk-professional-indemnity` |

## Document control

- Issue drawings/specs with revision code, date, and status (WIP / For Review / Issued).
- Supersede prior issues explicitly in transmittals.
- Maintain golden-thread / H&S file entries for HRB projects per project information standard.

## Artifact naming

| Artifact | Convention |
|----------|------------|
| Stage decision log | `SDL-[project]-[stage]-[YYYYMMDD]` |
| Compliance memo | `CM-[project]-[topic]-[rev]` |
| Snagging list | `SN-[project]-[zone]-[rev]` |

## Review gates

- **Stage 2→3:** Planning strategy locked; brief signed off.
- **Stage 4:** Building Control submission readiness; consultant coordination complete.
- **Stage 5→6:** Inspection regime, testing, PC checklist, Regulation 38 pack identified.
